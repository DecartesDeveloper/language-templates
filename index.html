Interface, helpers & the others…

This repository lists language templates used to power code samples in Apiary's API Documentations.

Please feel free to play around, contribute your own languages or improve the existing ones.

The HTML templates are written in ECT, which places data between <%= (escaped version), %<%- (original) and %> tags and control structures between <% and %>.

The following are variables and helper methods you can use when rendering the code of one request example:

  • @headers - a key/value Object (hash map) of HTTP headers to be sent
  • @body - a String containing the HTTP body to be sent
  • @method - HTTP method used in the request
  • @url - URL of the request, relative to the root of the server (ideally starts with '/')
  • @apiUrl - Hostname of the API mock/proxy server (includes protocol (http/https), does not include trailing '/')
  • @contentType - A shorthand for @headers['content-type'] (if applicable)
  • @helpers - key/value Object of helpers methods for additional use
    • @helpers.escape - escape " (double-quote) characters
    • @helpers.rubyKey - converts - into _ ruby friendly key (eg. morphs Content-Type into content_type
    • @helpers.getContentType(headers) - same as @contentType but in form of a function you can pass any array of headers
    • @helpers.getContentTypeBrush(headers) - produces a CSS class & enables syntax-highlighting (Prism) for this language
    • @helpers.isNotEmpty(obj) - same like [].length != 0 for Objects (hash maps)
